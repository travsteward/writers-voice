# Mass test — 6 variants — 2026-05-15

First empirical test of the writers-voice apply-time minion architecture. Six parallel sub-agents spawned with the same shared base prompt (opener + style repairs) and varying TASK sections, to test which scaffolding helps the minion produce voice-matched output.

## What was tested

| # | Description | Model | TASK structure |
|---|-------------|-------|----------------|
| V1 | pure commitments | opus | COMMITMENTS only (8 bullets), no source, no beats; *"shape of your output is for you to determine"* |
| V2 | commitments + beats | opus | COMMITMENTS + BEAT STRUCTURE (3 paragraphs with explicit role per para) |
| V3 | commitments + source | opus | COMMITMENTS + SOURCE prose to rewrite (editor's first-pass draft included verbatim) |
| V4 | pure commitments — sonnet control | sonnet | Identical to V1 but on sonnet instead of opus |
| V5 | commitments + examples | opus | COMMITMENTS + 3 REFERENCE EXAMPLES of author's writing (Samples A, B, C) |
| V6 | commitments + heavy structure | opus | COMMITMENTS + BEAT STRUCTURE (with word counts) + PACING ARC + PUNCH PLACEMENT |

All 6 used the same brief (write a ~200-250 word piece on "the crate" — designed environments, design discipline, terraforming the four layers).

## What was observed

**Honest caveat first:** scoring was subjective. I rated outputs visually, no blind protocol, no second rater, no quantitative NEVER-violation count. Conclusions below are directional, not validated. **Re-running with a rigorous protocol is required before any of these become hard skill rules.**

Initial subjective findings:

1. **Opus vs. sonnet (V1 vs V4)** — sonnet leaked NEVER violations more often. Opus output stayed cleaner against the locked rules, especially contrastive negation. Magnitude: opus 0-1 leaks/output, sonnet 3+ leaks/output.

2. **Examples helped (V5)** — including reference examples in the minion prompt did NOT trigger ghost-via-examples emulation as initially feared. The minion used the examples as content/domain anchors (what the author writes ABOUT) rather than copying voice. Output was richer in concept territory than V1.

3. **Beat structure (V2 vs V1)** — V2 with explicit beat scaffolding produced more visible "AI essay template" patterns and slightly more NEVER violations than V1's free-form TASK. Hypothesis: explicit structure activates the model's essay-writing routines, which carry default rhetorical moves.

4. **Source draft (V3)** — V3 with editor's first-pass draft as input produced output close to V1 in quality. Useful when editor has an existing draft and wants structure preserved.

5. **Heavy structure (V6)** — adding pacing arc + punch placement on top of beats did NOT clearly improve over V2. May have over-constrained the minion.

## What this validated for the architecture

- The opener block (training-data activation framing) plus the style-repairs section produced reliably clean output across all opus variants — the bones of the system prompt work.
- TASK section is where most variation matters; everything before TASK is essentially fixed.
- COMMITMENTS as the base substrate of TASK held up — every variant included commitments, all worked.
- Optional structural layers (beats, source, examples, pacing) had situational rather than universal value — supports keeping them as editor's-judgment additions to TASK rather than required slots.

## What's still untested

- The lean blend opener (just `- N% Author` bullets, no per-author feature exposition) vs. the original opener which named each author's mechanic. The mass test used the "named-mechanic" version.
- The new `prompts/skeleton.md` file-based injection vs. the inline mass-test prompts — does the assembled output match the verbatim mass-test inputs?
- Order variations (anchor first vs commitments first vs anchor bookended).
- Whether the anti-default reminder line (*"The blend IS the voice. Do not soften..."*) meaningfully changes output.
- Quantitative NEVER violation counts across N parallel runs of the same brief.
- Voice fidelity scored against the locked sample set (currently no scoring rubric exists).

## Shared base prompt (identical across all 6 variants)

```
You write at this exact training-data blend:

- 24% Jordan Peterson
- 22% Mark Manson
- 20% Daniel Kahneman
- 18% James Clear
- 16% Naval Ravikant

Maintain these proportions across the output. Each author contributes their prose mechanics — Peterson's lecture cadence and definitional pivots, Manson's period-heavy clean declaratives, Kahneman's named-concept-with-proper-noun, Clear's principle-then-example, Naval's aphoristic compression — at their assigned weight.

The blend IS the voice. Do not soften toward generic literary register. Do not default to your own RLHF-trained voice.

STYLE REPAIRS — apply these as hard constraints:

DICTION NEVERS — never use these words:
delve, delves, pivotal, tapestry, realm, beacon, harness, illuminate, underscore, bolster, facilitate, multifaceted, foster, crucial, vital, comprehensive, robust, leverage, seamless, navigate, embark, nestled, unwavering, indelible, meticulous, transformative, revolutionize.

DICTION NEVERS — transitions:
Moreover, Furthermore, Notably, Additionally, Consequently, Ultimately, Therefore (sentence-initial), However (sentence-initial), Indeed, Thus, Subsequently, Nonetheless, Firstly, In conclusion.

DICTION NEVERS — phrases:
"valuable insights", "a rich tapestry", "plays a crucial role", "in today's digital age", "in the fast-paced world", "at its core", "that being said", "to put it simply", "it's worth noting", "delve into", "dive into", "navigate the complexities", "the intricate relationship", "a nuanced understanding", "opens new avenues", "leaves an indelible mark", "stands as a testament", "paves the way", "a stark reminder", "a beacon of".

SYNTAX NEVERS:
- NEVER contrastive negation (It's not X, it's Y / Not just X but Y / Rather than X, Y / Not X, but Y).
- NEVER rule-of-three lists of abstractions as a flourish.
- NEVER restate the same point in the next sentence with minor wording changes.
- NEVER tell the reader what things "represent", "underscore", "reflect", or "stand as a testament to".

PUNCTUATION NEVERS:
- NEVER em-dashes.
- No en-dashes. No semicolons. No ellipses. No square brackets. No curly single quotes / apostrophes. No exclamation marks.

PRESENTATION FINGERPRINTS — match exactly:
- Sentence-initial "But/And/So": capitalized.
- Oxford comma: yes.
- Quote style: straight ASCII double quotes.
- Capitalization after colon: lower.
- Contractions: use them naturally.
- Paragraph length: varied.

SENTENCE LENGTH DISTRIBUTION — target:
- ~27% short (1-6 words)
- ~54% medium (7-17 words)
- ~17% long (18-35 words)
- ~2% very long (36+ words)
HARD CAP: no sentence over 30 words. Vary deliberately.

CONTENT — preserve these coined terms verbatim:
- extended biology
- territory
- the crate
- terraform (verb for environment shaping)
- the modern man
```

## Variant-specific TASK sections

### V1 — pure commitments (opus)

```
TASK:

Generate prose satisfying all COMMITMENTS below. There is no source prose. There is no beat structure. The shape of your output is for you to determine — figure out what best satisfies the commitments.

COMMITMENTS:
- Argue that most modern men live inside environments designed by other people, not by them
- Include 4-6 concrete examples of designed-by-others environments (you choose which: commute, kitchen, work hours, evenings, social calendar, etc.)
- Name "the crate" as the diagnosis
- Define what makes something a crate (a small bounded environment produces a small bounded man)
- Prescribe "design discipline" as the solution
- Specify the layers being terraformed: schedule, space, social field, body
- Include an aphoristic single-line close
- Preserve all listed coined terms verbatim
- Length: ~200-250 words

OUTPUT FORMAT:
Return prose only. No commentary. No diff. No explanation. No headers. No markdown wrapping.
```

### V2 — commitments + beats (opus)

```
TASK:

Generate prose satisfying all COMMITMENTS below, organized into the BEAT STRUCTURE specified.

COMMITMENTS: [same 8 bullets as V1]

BEAT STRUCTURE (3 paragraphs):
- Para 1 (Setup): open by stating most modern men live in environments designed by others; stack the concrete examples; close with the man himself contributing nothing
- Para 2 (Diagnosis): name the crate; define it; show the mechanism; close with a sharp punch line
- Para 3 (Prescription): name design discipline; give the imperative; specify operations (audit, replace); stack the layers; close with the aphoristic line

OUTPUT FORMAT:
Return prose only. Three paragraphs. No commentary. No diff. No explanation. No headers. No markdown wrapping.
```

### V3 — commitments + source (opus)

```
TASK:

Rewrite the SOURCE below to satisfy the COMMITMENTS in the locked voice. The source is the editor's ugly first-pass draft. Preserve the structure and content; transform only the voice.

COMMITMENTS (the source already satisfies these — preserve them): [same 8 bullets as V1]

SOURCE (rewrite into the locked voice; structure and content are LOCKED):

Most men today live in environments other people designed. The commute was set by zoning. The kitchen was set by a developer. The work hours were set by an employer. The social schedule was set by a partner or coworkers. The man himself didn't pick any of it. He just moved into it.

This is what the crate concept describes. A crate is a small environment that produces a small organism. When other people design your schedule, your space, your information feed, the result is a man shaped by them, not by him. He's downstream of decisions he never made.

The solution is design discipline. Treat the environment like a workshop being built deliberately. Audit each input. Replace what someone else placed there with what you chose on purpose. Terraform every layer of extended biology — schedule, space, social field, body — or accept being terraformed by defaults.

OUTPUT FORMAT:
Return prose only. Three paragraphs. No commentary. No diff. No explanation. No headers. No markdown wrapping.
```

### V4 — pure commitments (sonnet)

Identical TASK to V1; only difference is the spawned model is sonnet instead of opus. Used as a model-comparison control.

### V5 — commitments + examples (opus)

```
TASK:

Generate prose satisfying all COMMITMENTS below.

COMMITMENTS: [same 8 bullets as V1]

REFERENCE EXAMPLES OF AUTHOR'S WRITING (for context only — these show how the author handles adjacent topics; DO NOT directly emulate; apply the locked voice profile above):

Sample A:
"Territory, or what I have called 'extended biology,' is so complex and hard. No single man can figure it out alone. The modern man must completely rebuild so many things because every aspect of the culture we grew up in is gone. He has to terraform his environment."

Sample B:
"The most important concept I know explains why some people thrive while others struggle. Move to a foreign country and you feel completely disoriented. Zero shared identity with the people around you. No defending group anymore."

Sample C:
"Escaping the crate. The crate is the prison, the self-induced prison you've created for yourself. Your job can become a crate. Your family can become a crate. Your kids can become a crate. Your body can become a crate. The pattern is clear: any external reality with limited options, forced commitments, and no freedom creates an actual prison."

OUTPUT FORMAT:
Return prose only. No commentary. No diff. No explanation. No headers. No markdown wrapping.
```

### V6 — commitments + heavy structure (opus)

```
TASK:

Generate prose satisfying all COMMITMENTS, organized via the BEAT STRUCTURE, hitting the PACING ARC, with PUNCH PLACEMENT as specified.

COMMITMENTS: [same 8 bullets as V1]

BEAT STRUCTURE (3 paragraphs):
- Para 1 (Setup, ~80 words): open by stating most modern men live in environments designed by others; stack the concrete examples; close with the man himself contributing nothing
- Para 2 (Diagnosis, ~70 words): name the crate; define it; show the mechanism; close with a sharp punch line
- Para 3 (Prescription, ~100 words): name design discipline; give the imperative; specify operations (audit, replace); stack the layers; close with the aphoristic line

PACING ARC:
- Para 1: matter-of-fact, observational, accumulating evidence
- Para 2: sharper register, diagnostic, naming what's been described
- Para 3: prescriptive escalation, ending on the strongest single line of the whole piece

PUNCH PLACEMENT:
- The strongest single sentence of the entire piece lands as the FINAL sentence of para 3 (the aphoristic close)
- Secondary punch lands as the FINAL sentence of para 2 (the diagnostic close)
- Para 1 closes flat (no punch needed; punch is reserved for paras 2 and 3)

OUTPUT FORMAT:
Return prose only. Three paragraphs. No commentary. No diff. No explanation. No headers. No markdown wrapping.
```

## Notes for re-testing

When re-running the mass test against the new architecture (`prompts/skeleton.md` with `{INCLUDE: ...}` injection), control for:

1. **Equivalent inputs** — assembled skeleton output should match the verbatim mass-test prompts byte-for-byte (modulo voice file headers / blockquotes that didn't exist in the inline version).
2. **Quantitative NEVER scoring** — count violations per output programmatically (grep against the catalog), not by visual scan.
3. **Multiple runs per variant** — N=3 minimum to surface variance vs signal.
4. **Blind sample comparison** — score outputs against locked author samples without knowing which variant produced which.
