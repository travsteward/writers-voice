# Voice Profile

This directory is your **voice profile**. Files in here are read by the agent at write time and applied as style constraints. The skill builds these up progressively over time.

## What goes here

| File | Purpose | Source |
|------|---------|--------|
| `voice-match.md` | Anchor blend (3-5 training-data authors with weights) | Pasted from openwriter.io/voice-match |
| `stats.md` | Sentence distribution + punctuation density | Auto-generated by `extract.mjs` |
| `never-rules.md` | NEVER rules (kill-list) | Auto-generated + manual additions |
| `fingerprints.md` | Exact presentation choices (Oxford comma, etc.) | Auto-generated + manual overrides |
| `examples.md` | Curated reference paragraphs | Manually picked by the user |
| `status.md` | Current tier + what's locked | Auto-generated |
| `corpus/` | Raw samples accumulating over time | Manually added (drop files here) |

## How it grows

The more samples you accumulate in `corpus/`, the richer the auto-generated rules become. Tiers:

- **300-1k words**: anchor blend + basic stats
- **1k-5k words**: + preliminary NEVER rules + top fingerprints
- **5k-20k words**: + full NEVER coverage + all fingerprints
- **20k+ words**: AV-grade (high-confidence profile)

## Re-running extraction

After adding samples to `corpus/`, re-run:

```bash
node ~/.claude/skills/voice-anchor/extract.mjs ~/.claude/skills/voice-anchor/voice
```

Or just ask the agent: "regenerate my voice rules from corpus."

## Manual edits

`never-rules.md` and `fingerprints.md` have a `## Manual Additions` / `## Manual Overrides` section at the bottom. The script preserves anything you put there across re-runs.

## Privacy

This is your voice profile. The files live on your disk. The skill never uploads anything — extraction is local. The only thing that leaves your machine is the initial paste of writing into openwriter.io/voice-match for the anchor matching step (300-800 words, used once, cached 24h, never trained on).
